# Workflow OS 0.1.0-preview.1 local kernel preview

Workflow OS `0.1.0-preview.1` is the first public **local-first kernel preview**.

This preview is for builders, evaluators, and contributors who want to inspect the architecture, run the local vertical slice, exercise the validator and CLI, and help shape the next phases.

## What to try first

```sh
cargo build -p workflow-cli --bin workflow-os
target/debug/workflow-os --project-dir examples/vertical-slice-approval validate
target/debug/workflow-os --project-dir examples/vertical-slice-approval --mock-all-local-skills run ex/review
```

The example pauses for approval and prints a `run_id` and `approval_id`.

```sh
target/debug/workflow-os --project-dir examples/vertical-slice-approval --mock-all-local-skills approve <run-id> <approval-id> --actor user/example-approver --reason reviewed-local-example
target/debug/workflow-os --project-dir examples/vertical-slice-approval inspect <run-id>
```

## Included in this preview

- Declarative YAML project/spec loading.
- Rust-owned semantic validation.
- Event-sourced local run model.
- Local filesystem development backend.
- Conservative policy checks.
- Approval pause/resume.
- Retry, escalation, cancellation, and timeout representation.
- Audit and observability foundations.
- CLI commands for local validation, execution, approval, status, inspection, and doctor checks.
- TypeScript SDK helpers for spec generation.
- Schema/SDK/example contract checks.
- Generic adapter contracts for future integrations.

## Not included

This is not production-ready.

This release does not include:

- real GitHub, Jira, CI, generic HTTP, or SaaS adapters
- distributed workers
- production database backend
- hosted service
- UI
- marketplace or package registry
- active timeout scheduler
- trigger ingestion service
- enterprise RBAC, IdP, or secret-provider integration
- Level 3/4 autonomy enablement
- stable versioned CLI JSON contract

## Important preview notes

- Local mock skill execution is explicit through `--mock-all-local-skills`; arbitrary `local/*` skills do not run by default.
- CLI `--json` output is experimental in `0.1.0-preview.1`.
- `serde_yaml` is accepted for trusted local project files only in this preview, despite deprecation and transitive `unsafe-libyaml`. Workflow OS does not claim hardened malicious-spec parsing.
- Package publication is not automatic for this preview. The GitHub release source archive is the default public artifact unless maintainers separately approve crate or npm publication.

## Reporting issues

Please use the GitHub issue templates. Use the security policy for private vulnerability reports.

Good early feedback includes:

- confusing spec or CLI behavior
- validation diagnostics that should be clearer
- local state inspection issues
- schema/SDK mismatch concerns
- documentation overclaims or unclear limitations
- adapter contract design feedback

## What comes next

Next work should focus on release hygiene, local kernel hardening, schema/Rust/TypeScript synchronization, YAML parser replacement or isolation, and adapter readiness criteria. Real read-only adapters may come later, after the local-kernel preview posture is settled.
