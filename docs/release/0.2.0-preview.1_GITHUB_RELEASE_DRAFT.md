# Workflow OS 0.2.0-preview.1 — Public Read-Only Integration Preview

Workflow OS `0.2.0-preview.1` is a public preview of read-only integration capability for the local Workflow OS kernel.

This release demonstrates fixture-first and opt-in live read-only adapter contracts for:

- GitHub read-only context
- Jira read-only issue context
- GitHub Actions / CI read-only run context

It is **not production-ready**. It does not include writes, reruns, dispatches, webhooks, OAuth, hosted operation, distributed workers, production backends, generic live adapter execution, domain packs, production telemetry export, or Level 3/4 autonomy.

## What To Try

Build the CLI:

```sh
cargo build -p workflow-cli --bin workflow-os
```

Run fixture-backed examples:

```sh
target/debug/workflow-os --project-dir examples/github-read-only-review-context validate
target/debug/workflow-os --project-dir examples/github-read-only-review-context --mock-all-local-skills run ex/gh

target/debug/workflow-os --project-dir examples/jira-read-only-intake-quality validate
target/debug/workflow-os --project-dir examples/jira-read-only-intake-quality --mock-all-local-skills run ex/jira

target/debug/workflow-os --project-dir examples/ci-read-only-failure-summary validate
target/debug/workflow-os --project-dir examples/ci-read-only-failure-summary --mock-all-local-skills run ex/ci
```

Each example uses fixture-backed provider data and deterministic local mock handlers. They do not require live credentials and do not write to providers.

These examples pause for approval; use the RC1 Evaluation Guide for approve and inspect commands.

## Live Provider Posture

Live provider tests are opt-in maintainer checks, not normal CI. Use read-only credentials only and follow the live-smoke runbooks.

Rotate the Jira sandbox API token used during smoke testing before public promotion.

Recorded smoke limitations:

- GitHub live smoke exercised `octocat/Hello-World` repository metadata, not `rcs2153/AGT`.
- Jira live smoke exercised issue metadata only.
- GitHub Actions live smoke exercised workflow run metadata only.
- Jira sandbox token rotation is recommended because a sandbox token was pasted into the local evaluation thread before smoke execution.

## Reporting Issues

Please include:

- commit SHA
- command run
- fixture/mock/live mode
- redacted output summary
- expected vs actual behavior

Do not include tokens, authorization headers, private keys, full private issues, full pull requests, raw CI logs, or raw provider payloads.
