# Workflow OS 0.2.0-preview.1 Release Notes

Release title: **Workflow OS 0.2.0-preview.1 — Public Read-Only Integration Preview**

## Release Posture

Workflow OS `0.2.0-preview.1` is a public preview of read-only integration capability layered on the local Workflow OS kernel.

This is a preview for builders, evaluators, and contributors who want to evaluate how Workflow OS reads external context through governed adapter contracts.

This release is:

- read-only only;
- opt-in for live providers;
- fixture-first in normal CI;
- local/kernel-oriented;
- not hosted;
- not production-ready;
- not write-capable;
- not distributed-runtime-ready;
- not Level 3/4 autonomy-ready.

## What Is Included

- GitHub read-only adapter preview.
- Jira read-only adapter preview.
- GitHub Actions / CI read-only adapter preview.
- Fixture-backed tests and examples for all three provider families.
- Opt-in maintainer live smoke test wrappers for GitHub, Jira, and GitHub Actions.
- Adapter policy-precheck provenance, with fixture/test authorization distinct from runtime policy authorization.
- Local runtime-visible adapter telemetry mapping for controlled fixture-backed examples.
- Provider setup, token-scope, troubleshooting, live-smoke, and integration testing documentation.
- Recorded live smoke evidence for one narrow read path per provider family.

## What Is Not Included

This release does not include:

- GitHub writes: branch creation, commits, pull request creation, comments, reviews, labels, merges, or closes.
- Jira writes: issue creation, issue updates, comments, transitions, assignment, labels, or links.
- CI writes: workflow reruns, failed-job reruns, workflow dispatch, cancellation, artifact mutation, log deletion, or check mutation.
- Generic live adapter execution from arbitrary workflow specs.
- Generic HTTP adapter execution.
- Webhooks or event ingestion services.
- OAuth app flows.
- Hosted integration service.
- Production database backend.
- Distributed workers.
- Production telemetry export, SIEM integration, or OpenTelemetry export.
- Domain packs.
- Level 3/4 autonomy enablement.

## Provider Scope

### GitHub Read-Only

Implemented read-only contract paths include repository metadata, default branch, file content references, pull request metadata, pull request diff summaries, changed files, comments as read-only data, and check-run summaries.

Accepted live-smoke limitation: the recorded GitHub live smoke exercised repository metadata for `octocat/Hello-World`, not the approved `rcs2153/AGT` repository. Broader GitHub operation coverage is fixture-tested, not live-proven in this release.

### Jira Read-Only

Implemented read-only contract paths include issue metadata, summary, description references, comments as reference-only data, status, priority, labels, assignee/reporter display metadata, and project metadata.

Accepted live-smoke limitation: the recorded Jira live smoke exercised issue metadata only against a sandbox issue. Jira descriptions, comments, and project metadata are fixture-tested, not live-proven in this release.

Security note: rotate the Jira sandbox API token used during smoke testing because a sandbox token was pasted into the local evaluation thread before smoke execution.

### GitHub Actions / CI Read-Only

Implemented read-only contract paths include workflow run metadata, job summaries, check summaries, failure summaries, log references, and bounded redacted log excerpts.

Accepted live-smoke limitation: the recorded GitHub Actions live smoke exercised workflow run metadata only for `rcs2153/AGT` run `26415289853`. Jobs, check summaries, failure summaries, log references, and redacted log excerpts are fixture-tested, not live-proven in this release.

## Fixture-First CI

Normal CI remains offline and credential-free. `npm run check:integrations` verifies read-only adapter contracts and fixture-backed examples without live provider credentials.

Live tests are opt-in maintainer checks only. They must not run in normal CI.

## Opt-In Live Provider Posture

Live provider tests require explicit environment variables and read-only credentials. See:

- [GitHub read-only setup](../operations/github-read-only-setup.md)
- [Jira read-only setup](../operations/jira-read-only-setup.md)
- [GitHub Actions read-only setup](../operations/github-actions-read-only-setup.md)
- [Maintainer live smoke tests](../integrations/live-smoke-tests.md)

Do not put credentials in Workflow OS specs, fixtures, examples, docs, screenshots, issues, or pull requests.

## Security And Privacy Notes

- Credentials are loaded from environment variables only.
- Specs must not contain provider tokens.
- Recorded smoke evidence does not include token values, authorization headers, raw provider payloads, raw issue bodies, full pull request content, or raw CI logs.
- Provider metadata can still be sensitive even when read-only.
- Redaction is deterministic preview redaction, not enterprise DLP.
- Adapter summaries prefer references, summaries, size metadata, and redaction metadata over raw sensitive payloads.
- Treat private repository, Jira, and CI metadata as sensitive.

## Adapter Telemetry Posture

Adapter invocation and observability records can be mapped into local runtime-visible adapter telemetry records for controlled fixture-backed examples.

This is not:

- generic adapter execution;
- live provider execution by default;
- production telemetry export;
- SIEM integration;
- OpenTelemetry integration.

## Upgrade / Migration From 0.1.0-preview.1

No persisted-state migration is introduced by this release posture pack.

Users evaluating `0.1.0-preview.1` local-kernel workflows can continue to use the local kernel commands and vertical slice. `0.2.0-preview.1` adds read-only integration preview surfaces and examples, but does not convert existing projects into live provider workflows automatically.

If local state was created by earlier preview builds, keep it for evidence only and prefer fresh temporary state directories for preview evaluation.

## Known Limitations

See [V0_KNOWN_LIMITATIONS.md](V0_KNOWN_LIMITATIONS.md) for the broader v0 limitation set.

Additional `0.2.0-preview.1` limitations:

- Live smoke coverage is intentionally shallow.
- Live tests remain maintainer-owned and opt-in.
- Normal CI does not require or verify live credentials.
- Generic live adapter execution from workflow specs is not implemented.
- Adapter telemetry is local/runtime-preview telemetry only.
- Write-capable adapters remain deferred.

## Next Planned Follow-Ups

- Add broader opt-in live smoke coverage for GitHub pull request/file/check paths.
